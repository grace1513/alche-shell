# Processes and Signals

This project focuses on understanding how processes and signals work in a Linux environment using Bash scripting. In Unix-like systems, a process is an instance of a running program, and each process is identified by a unique Process ID (PID). Signals are a form of inter-process communication used to control or manage these processes (for example, stopping, continuing, or terminating them).

Through this project, we explore how to interact with processes using simple Bash scripts. This includes retrieving process information such as the PID of a running script and preparing for more advanced tasks involving process management and signal handling.

By completing this project, we gain a stronger understanding of how the operating system manages running programs and how developers can interact with them efficiently using shell commands and scripting techniques.
